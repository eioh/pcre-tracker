# Lessons

## 2026-07-13: 調査はローカルチェックアウトだけでなく origin/main と最近のPRも見る

- **何が起きたか**: 「キャラデータの編成順を使う」という依頼に対し、ローカルチェックアウト(古いフィーチャーブランチ)だけを調査して「編成順フィールドは存在しない」と報告した。実際には PR #8 で `formationOrder` が main にマージ済みで、ユーザーに2回指摘されて判明した。
- **ルール**: ユーザーが「データ/機能がある」と言ったら、ローカルに見つからなくても即座に否定しない。`git fetch` して origin/main との差分・最近のマージPR (`gh pr list --state merged`) を確認してから結論を出す。作業ブランチが main より遅れている可能性を常に疑う。
